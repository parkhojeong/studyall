logical modeling 
- goal
  - data logical structure definition
  - transform business logic into relational model
- main works
  - entity -> table
  - column definition(abstract data type, pk, fk)
  - relationship definition(1:1, 1:n, n:n)
  - normalization(1nf ~ bcnf)
  - constraint definition(not null, unique, etc)
- consideration
  - dbms independent
  - data integrity
  - minimization of redundant data
- tool/output
  - erd
  - erd to rdb schema
  - ddl script 1st version
  - term dictionary

physical modeling
- goal
  - implementation optimized for specific dmbs
  - considering performance, storage
- main works
  - table, column, data type definition
  - table space, file group allocation
  - index creation(clustered/non-clustered, b-tree, etc)
  - partitioning design(range, hash, etc)
  - storage engine selection
  - view, trigger, stored procedure definition
  - security definition(access control, encryption)
  - performance tuning(query optimization, caching strategy)
- consideration
  - dbms dependent
  - hardware resources(cpu, memory)
  - analysis query pattern and workload
  - scalability and backup/recovery
- tool/output
  - final ddl script(including partitioning, index)
  - table definition report
  - physical diagram
  - performance test script

* physical modeling process
1. table, column transformation
2. data type definition
3. constraint definition
4. index definition
5. denormalization and performance tuning
6. partitioning, sharding, etc
7. view, trigger, stored procedure definition
