* database design first step 

* lecture flow
1. design importance: why design is important, how wrong design makes disaster
2. design 3 steps roadmap: understand the big picture  
  - processes making a database from idea(conceptual -> logical -> physical)
3. conceptual/logical modeling
  - by analyze the requirement, find core data(entity)
  - define properties of each entity
  - represent concisely relationship between entities
  - detail
    - how to writing Entity-Relationship Diagram
    - various key types and roles
    - relationship types(1:N, M:N)
    - identifying/non-identifying relationship
4. normalization: increase model quality by remove duplication and data and increase data integrity
5. physical modeling: how to make optimized table to MySQL database from the logical model
