* entity?
- entity is target to store and manage
- more detail, tangible and intangible things that have valuable information and distinguish from others

* tangible entity
- targets physical shaped like human, object, place
  - human, employee
  - product, book
  - building

* intangible entity
- targets are not physical but conceptual. mainly any 'event' or 'concept'
  - order, reservation, class application: event
  - account, department: concept

* good entity characteristics
1. business relevance: used for business purpose
2. uniquely identifiable: each row has unique
3. has attributes: has two or more columns
4. set of instance
5. has relationship (sometimes hasn't relationship)

* entity and classification of property
- confusion between entity and property
- does this independently have to be managed?
  - yes -> entity
  - no -> property
