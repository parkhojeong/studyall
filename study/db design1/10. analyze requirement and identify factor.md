* analyze requirement and identify factor

* shopping mall requirement
- user function
  - user can join with unique id, password, name, address(basic delivery address), concact
  - user can edit own information
  - user can leave system
- product function
  - admin(or seller) can add product
  - product has unique product code, name, price, stock quantity
  - product's name or stock quantity can be updated anytime
- order function
  - user can order multiple products at once
  - when order product, each product quantity can be selected
  - when order product, user can change address(it can be different from basic delivery address)
  - when order complete, order status will be '주문 완료', and unique order code is created
  - order date, final payment price, delivery address will be saved
  - admin can update order status to '배송 중', '배송 완료'
  - user can query order history as a list

- derived entity
  - user
  - product
  - order

* focus on nouns and verbs
- nouns are a high chance to be an entity or attribute
- verbs are a high chance to be a relationship 

* core data, extract entity
- entity has important data to save
- entity is a set of each data. ex. 'nate', 'sean'

* order entity
- finding nouns is a guideline to easily extract entity candidates
- order is a verb, but we store order history(record)
- sellers want what and when the user ordered

* rule of finding nouns is actual meaning
1. nouns are candidates of entity
2. find important verb(order, reserve)
3. check verb has history that should be stored
4. if true, make an entity from the noun of verb
