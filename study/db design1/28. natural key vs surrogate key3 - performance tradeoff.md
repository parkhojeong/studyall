* natural key vs surrogate key3 - performance tradeoff

* natural key pros and cons
- pros 
  - query simplicity: join can be omitted
- cons
  - type size fat
    - space wasted: use more disk space
    - performance dropped 
  - index fragmentation and write performance dropped
    - frequent page split
    - fragmentation increase
  
  
* artificial key pros and cons
- pros
  - small type size
  - high write performance
    - auto_increment/sequence needs only append to generate new value
    - less page split
    - MySQL support clustering index
- cons
  - additional join required
  - additional index required
    - pk index and natural key unique index
