* one-to-one relationship - fk position

- where to put fk
1. in main tale: main -> detail(recommended)
  - pros
    - checking detail existence is easier
    - the main table has detail
    - not null constraint on detail: optionality is clear
  - cons
    - dependency: main dependent on detail
    - low flexibility: hard to change detail 1:1->1:n
    - row can have many nulls
2. in detail table: main <- detail (recommended)
  - pros
    - main does not depend on detail
    - logical clarity: detail depends on main
    - show optional relationship: ex. all members dose not have a detail
    - flexibility to change detail 1:1->1:n: only eliminate unique
  - cons
    - checking detail existence
