```sql
create table product(
    ...
                    
    change_type varchar(50),
    change_reason varchar(500),
)
```
- change_type: change type(PRICE_CHANGE, STATUS_CHANGE, STOCK_ADJUSTMENT)
- change_reason: change reason (free format text)

price change example
```sql
update product
set price = 12000,
    updated_by = 'admin_park',
    change_type = 'PRICE_CHANGE',
    change_reason = 'discount event for spring season(2026-03-01 ~ 2026-03-31)'
where product_id = 1;
```

status change example
```sql
update product
set status = 'INACTIVE',
    updated_by = 'admin_lee',
    change_type = 'STATUS_CHANGE',
    change_reason = 'discontinued by manufacturer'
where product_id = 2;
```

change type standardization
example

| change_type   | description                                |
|---------------|--------------------------------------------|
| CREATE        | new creation                               |
| PRICE_CHANGE  | price change                               |
| STOCK_ADJUST  | stock adjustment                           |
| STATUS_CHANGE | status changed                             |
| INFO_UPDATE   | information update(name, description, etc) |
| PROMOTION     | promotion applied                          |
| correction    | error correction                           |

* why change_type is required?
for good use and easy to search
1. structured data
it makes exact separation between a change type
2. easy to search
- where change_type = 'PRICE_CHANGE'
- if change_type is exist, where change_reason like '%stock%' or '%promotion%' or '%discount%'...
