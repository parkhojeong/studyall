* requirement: weakly_sales_stats, monthly_sales_stats
- daily_sales_stats can quickly make weakly and monthly stats
- only 365 rows exist in a year

```sql
use my_shop5;
insert into daily_sales_stats values('2025-11-01', 1, 10000)
insert into daily_sales_stats values('2025-11-02', 2, 15000)
insert into daily_sales_stats values('2025-12-01', 2, 20000)
insert into daily_sales_stats values('2025-12-02', 3, 21000)
```

- requirement: monthly_sales_stats(2025-11~2026-01)
```sql
select 
    date_format(stat_date, '%Y-%m') as stat_month,
    sum(total_order_count) as monthly_order_count,
    sum(total_sales_amount) as monthly_sales_amount
from daily_sales_stats 
where stat_date >= '2025-11-01' 
  and stat_date < '2026-02-01'
group by date_format(stat_date, '%Y-%m')
```

- requirement: yearly_sales_stats(2025~2026)
```sql
select 
    date_format(stat_date, '%Y') as stat_month,
    sum(total_order_count) as monthly_order_count,
    sum(total_sales_amount) as monthly_sales_amount
from daily_sales_stats 
where stat_date >= '2025-01-01' 
  and stat_date < '2027-01-01'
group by date_format(stat_date, '%Y')
```

* summary
- original table rows(n counts) -> daily stats rows(365 counts in a year)
- daily stats can quickly make weakly and monthly stats
- in huge scale, monthly stats can be considered
