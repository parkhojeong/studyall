- create
  - _at : when created
  - _by : who created
- updated
  - _at : when updated
  - _by : who updated

pros
1. design is simple
2. application code is simple


```sql
create table product(
    product_id bigint primary key auto_increment,
    created_at datetime not null default current_timestamp,
    created_by varchar(255) not null,
    updated_at datetime not null default current_timestamp on update current_timestamp,
    updated_by varchar(255) not null
)
```

question list
1. who is changed? -> solved
2. when is changed? -> solved
3. why it changed? -> not yet
4. what is previous value? -> not yet

* when is it used?
1. master data that history is not required: code table, setting table
2. internal admin data
3. admin log purpose

* practice tips
- a change tracking column is used in almost table
  - created_at, created_by, updated_at, updated_by
  - when data never changed, updated is omitted
