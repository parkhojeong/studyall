* javascript object notation - de facto standard in web

basic structure
- object: wrapped by {}, key-value pairs, key must be wrapped by ""
```json
{
  "key": "value",
  "key2": 123
}
```
- array: wrapped by []
```json
["value1", "value2"]
```

json data type
- string: wrapped by "", '(single quote) cannot be allowed
- number: integer or floating point
- boolean: true/false. it does not be wrapped by ""
- null: lowercase
- object: value can be any type of data
- array: it can contain any type of data

practice popular json pattern
- product attribute 
```json
{
  "screen_size": 6.7,
  "storage": 256,
  "ram": 8,
  "color": "midnight blue",
  "5g_support": true,
  "release_data": "2026-03-15"
}
```
- internationalization support
```json
{
  "name": {
    "ko": "무선 이어폰",
    "en": "wireless phone"
  }
}
```
- option and variants
```json
{
  "variants": [
    {"color":  "black", "size":  "M", "stock":  50, "sku":  "BLK-M-001"}
    {"color":  "black", "size":  "L", "stock":  30, "sku":  "BLK-L-001"}
    {"color":  "white", "size":  "M", "stock":  50, "sku":  "WHT-M-001"}
  ]
}
```
- metadata
```json
{
  "seo": {
    "title": "galaxy s22 Ultra",
    "description": "galaxy s22 Ultra 5G | Samsung",
    "keywords": ["galaxy", "s24"]
  }
}
```

* json usage caution
- key must be double-quoted
```json 
{
  'key': 'value' 
}
```
- trailing comma is not allowed
```json
{
  "key": "value",
}
```
- comment is not allowed
```json
{
  "key": "value" // comment
}
```
- special characters(", \) can be used with escape sequence(\)
```json
{
  "key": "value is \"double quoted\"",
  "path": "c:\\temp\\file.txt"
}
```
