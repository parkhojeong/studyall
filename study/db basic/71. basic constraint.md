* basic constraint

* NOT NULL: prevent null
* UNIQUE: prevent duplicate
* PK: primary identifier to row
  - role: identify each row uniquely. NOT NULL + UNIQUE
  - characteristic: only one per table. important role for performance
    - MySQL 
      - automatically creates a high-performance index for pk
      - provide AUTO_INCREMENT: insert data with NULL, input automatically
* DEFAULT
  - characteristic: default value. strictly not a rule, just easy to use.

```sql
create table users (
    user_id bigint auto_increment, -- auto_increment
    name varchar(255) not null, -- not null
    email varchar(255) not null unique, -- unique
    address varchar(255) default 'Seoul', -- default 
    primary key (user_id) -- pk
);

insert into users (name, email) values ("abc", null);
-- Error Code: 1048. Column 'email' cannot be null

insert into users (name, email) values ("abc", "a@a.com");
insert into users (name, email) values ("abc", "a@a.com");
-- Error Code: 1062. Duplicate entry 'a@a.com' for key 'users.email'

insert into users(user_id, name, email) values (1, '2', '2');
insert into users(user_id, name, email) values (1, '2', '2');
-- Error Code: 1062. Duplicate entry '1' for key 'users.PRIMARY'
```
