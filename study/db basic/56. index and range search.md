* index and range searching

* without index
```sql
EXPLAIN SELECT * FROM items WHERE price BETWEEN 50000 AND 100000;
```
```
+----+------+------+------+----------+-------------+
| id | type | key  | rows | filtered | Extra       |
+----+------+------+------+----------+-------------+
| 1  | ALL  | NULL | 25   | 11.11    | Using where |
+----+------+------+------+----------+-------------+
```
- `type: ALL` - full table scan
- `key: NULL` - no index used
- `rows: 25` - number of rows in the table
- `filtered: 11.11` - percentage of rows that satisfy the condition
- `Extra: Using where` - after read data from the table, filter using the WHERE clause

```sql
SELECT * FROM items WHERE price BETWEEN 50000 AND 100000;
```
```
+---------+-----------+-----------------------+------------+-------+----------------+
| item_id | seller_id | item_name             | category   | price | stock_quantity |
+---------+-----------+-----------------------+------------+-------+----------------+
| 3       | 2         | 프리미엄 게이밍 마우스      | 전자기기     | 80000 | 200            |
| 6       | 5         | 고급 가죽 지갑            | 패션       | 70000  | 120           |
| 8       | 6         | 캐시미어 스웨터           | 패션        | 95000 | 70             |

```
- result sorted by item_id, because the order of physical saved is sorted
- db not guarantee the order of result. if you want sorted result, you should use `ORDER BY` clause

* with index
`CREATE INDEX price_idx ON items(price);`

```
+----+-------+-----------------+------+----------+-----------------------+
| id | type  | key             | rows | filtered | Extra                 |
+----+-------+-----------------+------+----------+-----------------------+
| 1  | range | idx_items_price | 5    | 100.00   | Using index condition |
+----+-------+-----------------+------+----------+-----------------------+
```
- `type: range` - range scan by index
- `key: idx_items_price` - index name
- `rows: 5` - number of rows in the table
- `filtered: 100.00` - use all of 5 rows  
- `Extra: Using index condition` - only with index, read all data satisfying where condition

* index range scan logical execution flow
1. find the first row that satisfies the condition(>= 50000)
2. index is sorted by price, read next row while condition is satisfied(<= 100000)
3. if row is over 100000, stop reading
