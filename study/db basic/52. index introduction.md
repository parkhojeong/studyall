* index introduction

* index 
- core purpose: increase search performance, query performance optimization
- definition: store specific column sorted, and access original data fast. 
- store pair 
  - selected column(1) 
  - original data record position(2)
    - ex. memory value, pointer, pk value, ...
- index internal data keep always sorted
  - fast search

* select execution flow using index
`SELECT * FROM items WHERE item_name = 'apple'`
1. instead of full table scan, find item_name index
2. index is sorted, fast search 'apple'
3. get record position from index
4. get original data record from position

* index implementation
- normal and logical index concept
- ex. MySQL provide clustered index, secondary index
  - clustered index: index based on pk, store original data with index together
  - secondary index: store pk with index. and via clustered index by pk, search data
