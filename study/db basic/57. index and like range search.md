* index and like range search

```sql
explain select * from items where item_name like '게이밍%';
```
```
+----+-------+---------------------+------+----------+-----------------------+
| id | type  | key                 | rows | filtered | Extra                 |
+----+-------+---------------------+------+----------+-----------------------+
| 1  | range | idx_items_item_name | 2    | 100.00   | Using index condition |
+----+-------+---------------------+------+----------+-----------------------+
```
- idx_items_item_name is sorted by item_name
- find a row starting with '게이밍' and search next repeatly

- like '%게이밍%': full table scan because sorted data can't be used

* practical tip: full-text search
- like '%keyword%' : performance issue
- db support full-text search. index using split text into word(token) unit
  - for full-text search, use `MATCH ... AGAINST` instead of LIKE 
  - other alternatives: elastic search
