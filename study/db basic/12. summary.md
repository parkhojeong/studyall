- JOIN requirment
  - split table by normalization
  - combine table by join
  - join is core technique that combine two table by fk and pk
- inner join 1
  - if data is present for both table, show that row
  - include row if on condition is true
  - join logical execution order combine by FROM/JOIN, filter by WHERE, choose column by SELECT
  - INNER keyword can be omitted
- inner join 2
  - find intersection of two table
  - A JOIN B = B JOIN A
- inner join 3
  - alias table name(recommended). without AS
  - alias column name with AS
  - inner join order not important
