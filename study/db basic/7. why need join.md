| order_id | order_date | user_name | user_email | product_name              | price   | quantity |
+----------+------------+-----------+------------+---------------------------+---------+----------+

problems to all in one table, 
1. redundancy
  - suppose 'sean' order 100 times
  - -> email, name duplication 100 times, waste of space
2. update anomaly
  - suppose 'sean' change email
  - -> update all email 100times
3. insertion anomaly
  - suppose create new product 'ligth weight notebook'
  - -> but, can't insert new column
4. deletion anomaly
  - suppose delete order of 'nate'
  - -> delete info of 'nate'

after normalization, split all in one table
- order table
- user table
- product table

join is used to combine data from multiple tables
- orders(user_id) -> user(user_id), fk -> pk
- orders(product_id) -> product(product_id), fk -> pk
