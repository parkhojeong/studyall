* index design guideline
- where to create index

* core principle: cardinality
uniqueness of values that column contains
- high cardinality: rarely duplicated values
  - ex. items table's item_id, item_name
- low cardinality: frequently duplicated values
  - ex. items table's category(5 type), is_active(2 type)
core principle: create index on columns with high cardinality

* index creation guideline
# 1. consider column in where clause
  - ex. if search by item_name, filter by category  
# 2. join condition column(fk)
```sql
select
    s.seller_name,
    i.item_name,
    i.price
from items i 
join sellers s on i.seller_id = s.seller_id
where s.seller_name = '행복쇼핑';
```
- execution flow if index(items.seller_id) not exists
1. find seller_name = '행복쇼핑' (seller_id = 1)
2. find selle_id = 1 by full scan

- execution flow if index(items.seller_id) exists
1. find seller_name = '행복쇼핑' (seller_id = 1)
2. find seller_id = 1 by index(items.seller_id)

* practice tip
- must create index to fk column 
- difference of join logical order and physical order
  - sql logical order is from join and where
  - sql optimizer: where(reduce data) -> join
