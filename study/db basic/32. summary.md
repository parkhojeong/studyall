* summary

- subquery type and characteristics
- single column
  - single row - =, >, <
  - multiple rows - IN, ANY, ALL
- multiple column
  - single row - (c1, c2) = ... 
  - multiple rows - (c1, c2) IN
- in from clause
  - called: table subquery, inline view

- subquery introduce
  - "select query" in sql
  - row(>= 0), column(>= 1)
- scalar subquery
  - single row and single column
  - must be scalar. if not, error occurs
- multi row subquery
  - single column and multiple rows
- multi column subquery
  - two or more columns
- correlated subquery1
  - repeat each row
- correlated subquery2
  - usage: EXISTS
- SELECT subquery
- TABLE subquery
  - when used in FROM clause
  - must has alias
- subquery vs join
  - consider join first
