* index creation, check, delete

* CREATE INDEX: index creation
```sql
CREATE INDEX index_name ON table_name(column1, column2, ...);
```
* practice index naming: recommend idx_tablename_columnname

* index check
`SHOW INDEX FROM table_name`

* key_name: index name
* column_name: index based on what column
* PRIMARY, seller_id
  - MySQL auto create index for pk, fk, unique
* Non_unique: allow duplicate value
* cardinality: unique estimation value
  - as it is higher, duplication rate is lower

* why auto create index for unique constraint?
- when insert new data, must check duplication. if the index doesnâ€™t exist, full table scan.
- if index exist, check duplication is fast.

* DROP INDEX: index deletion
`DROP INDEX index_name ON table_name`

* EXPLAIN: actually that index is used or not
```
+----+-------+-------+------+----------+----------------+
| id | type  | key   | rows | filtered | Extra          |
+----+-------+-------+------+----------+----------------+
| 1  | ALL   | NULL  | 25   | 10.00    | Using where    |
+----+-------+-------+------+----------+----------------+
```
- `type: ALL` means full table scan
  - when use index, ref(=, JOIN index), range(BETWEEN, >, <, etc)
- `key: NULL` means no index used
- `rows: 25` means row count estimation to search
- `filtered: 10.00` means filtered percentage estimation in 25 rows
- `Extra: Using where` means using where condition
- conclusion: full table scan and comparing with where condition
