# remind: composite index general rule
1. use index sequentially(index left-prefix rule)
2. = condition first, range condition last
3. order by should follow index order

* composite failure example1: ignore index order(rule 1)
```sql
explain select * from items where price = 80000;
```
- full table scan
- index is not sorted by price only

* composite failure example2: use range first(rule2)
```sql
explain select * from items where category >= '패션' and price = 20000;
```
```
+----+-------+---------------------------+------+----------+-----------------------+
| id | type  | key                       | rows | filtered | Extra                 |
+----+-------+---------------------------+------+----------+-----------------------+
| 1  | range | idx_items_category_price  | 6    | 10.00    | Using index condition |
+----+-------+---------------------------+------+----------+-----------------------+
```
- if the range condition is used, the following column can't properly use index
- `filtered: 10` 
  - category range is properly used, but filter each row for price comparison(=)
- summary logical execution plan
  1. using idx_items_category_price, find category = '패션' position
  2. scan from the position to the end in index
  3. filter each row to 2 stage results for price comparison(=)
- conclusion: if range condition is used, then following data is not sorted
