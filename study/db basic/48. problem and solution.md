* problem and solution

* problem1: view creation that show user basic info
- from users, only show user_id, name, email
- create v_customer_email_list view
```sql
create view v_customer_email_list as
select user_id, name, email
from users;
```

* problem2: order detail info view
- order_id, user_name, product_name, order_quantity, order_status
- create v_order_summary view
```sql
create view v_order_summary as
select 
	o.order_id, 
    u.name as user_name, 
    p.name as product_name, 
    o.quantity as order_quantity,
    o.status as order_status
from 
	orders o
    join users u on o.user_id = u.user_id
    join products p on o.product_id = p.product_id;
```

* problem3: '전자기기' category sales analysis view  
- from products, orders
- by category = '전자기기'
- total_orders, total_sales
- create v_electronics_sales_status view
```sql
create view v_electronics_sales_status as
select 
    '전자기기', 
    count(*) as total_orders, 
    sum(o.quantity * p.price)
from orders o
join products p on o.product_id = p.product_id
where p.category = '전자기기'
group by p.category;
```

* problem4: alter view to add new column
- to problem3 view, add average_order_value column
```sql
alter view v_electronics_sales_status as
select 
    '전자기기', 
    count(*) as total_orders, 
    sum(o.quantity * p.price),
    avg(o.quantity * p.price) as average_order_value
from orders o
join products p on o.product_id = p.product_id
where p.category = '전자기기'
group by p.category;
```
