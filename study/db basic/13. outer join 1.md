* practice: user that don't have order
1. find user: select * from user; // 1~6
2. find order: select user_id, order_id from orders order by user_id; // 1~5
- compare result of 1, 2 

* outer join requirement
- include only user that don't have order
- concept
  - users(left): [6, 1,2,3,4,5]
  - orders(right):   [1,2,3,4,5, 7] 
  - [6 / 1,2,3,4,5 / 7]
  - left outer join: [6 / 1,2,3,4,5] / 7 
  - right outer join: 6 / [1,2,3,4,5 / 7]

* LEFT OUTER JOIN vs RIGHT OUTER JOIN
- base table is important. LEFT OUTER JOIN base on left table
- practice: recommend OUTER omitted
- LEFT JOIN(LEFT OUTER JOIN)
  - left table is base table
  - include all data of left table
  - and attach data of right table if on condition is true
  - if on condition is false, attach null

* filter row - order_id is null
```sql
select
    *
from users u 
left join orders o on u.user_id = o.user_id
where o.order_id is null;
```
