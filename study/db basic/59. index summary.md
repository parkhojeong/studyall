* index summary
- index requirement: avoid full table scan, and fast query
- index introduction: index structure is sorted always
- tree data structure: b-tree, O(log n)
- index usage 
  - command: create index, show index, drop index 
  - auto index creation for pk, fk, unique
  - explain command: check execution plan
  - type: ALL - full scan, range,ref - index scan
- index and equality comparison
  - when index is used for '=', type is ref 
- index and range search
  - when index is used for range(between, <, >), type is range 
- index and like search
  - should end with %, ex. 'abc%'
  - if start with %, full scan. ex, '%abc'
  - for %keyword%, use full-text search instead index 
- index and order by
  - filesort is expensive operation 
  - by index, omit filesort
  - if where condition and order by condition are the same as index, filesort can be omitted
  - when order by desc, backward index scan omit filesort
