* tree data structure
- tree has parent node, child node
- node store data and next node
- top node is root
- if a node has child node up to 2, it is a binary tree
- if left child < parent < right child, it is a binary search tree

* binary search tree - example
input - 10,5,15,1,6,11,16

     10

* binary search tree bigO - O(logn)
- n = 2 -> log2(2) = 1
- n = 4 -> log2(4) = 2
- n = 8 -> log2(8) = 3
- ...
- n = 1024 -> log2(1024) = 10
- n = 1,000,000 -> log2(1,000,000) ~= 20
- n = 100,000,000 -> log2(100,000,000) ~= 26

* in order traversal flow
- left -> root -> right
- ex. 1,5,6,10,11,15,16 -> ascending order

* balanced tree 
- bst input: 1,5,6,10
```
  1
   \
    5
     \
      6
       \
        10
```
- worst: O(n)
- balance dynamically: balanced tree O(log n)

- B-Tree

  - many child nodes - for disk i/o count decrease
