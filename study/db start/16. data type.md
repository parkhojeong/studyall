* data type
- efficiency: save disk space
- correctness: prevent error

# number type(Bytes)
  - INTEGER, INT(4)
  - TINYINT(1)
  - SMALLINT(2)
  - MEDIUMINT(3)
  - BIGINT(8)
  - DECIMAL(M,D), NUMERIC(M,D): M- max digits, D- decimal digits
  - BIT(M): store bit
* UNSIGNED: starting with 0
* Approximate value numeric types
  - FLOAT(4)
  - DOUBLE(8)

# string type
  - VARCHAR(N): max length N
  - CHAR(N): fixed length N. padding space
  - TEXT: max length 65,535
    - MEDIUMTEXT
    - LONGTEXT

* VARCHAR vs CHAR 
- storage
  - VARCHAR: meta data + data
  - CHAR: fixed length + padding space
- max length
  - VARCHAR: 65,535 bytes
  - CHAR: 255 bytes
- space efficiency
  - VARCHAR: efficient use 
  - CHAR: wasteful
- performance(un meaningful)
  - VARCHAR: data fragmentation, page splitting
  - CHAR: fast update
- practical: use VARCHAR
  - space efficiency
  - flexibility
  - if data length is completely fixed, use CHAR
    - ex. resident registration number, country code, postal code, MD5 hash(32 length)

# date and time type
- DATE: YYYY-MM-DD
- TIME: HH:MM:SS
- DATETIME(5): YYYY-MM-DD HH:MM:SS
  - '1000-01-01 00:00:00.00000' ~ '9999-12-31 23:59:59.99999'
- TIMESTAMP(4): DATETIME + Timezone auto conversion
  - '1970-01-01 00:00:01.0000' ~ '2038-01-19 03:14:07.9999'
  - when save, convert UTC by server timezone
  - when read, convert by client timezone
- YEAR: YYYY

* DATETIME vs TIMESTAMP
- Timezone process
  - DATETIME: save and read value as is
  - TIMESTAMP: save value to db timezone UTC and read value to client timezone
- range: DATETIME
  - DATETIME: 1000~9999 year
  - TIMESTAMP: 1970~2038 year
- storage:
  - DATETIME: 5 bytes + alpha
  - TIMESTAMP: 4 bytes + alpha
- auto initialize/update
  - DATETIME: since MySQL 5.6.5
  - TIMESTAMP: insert/update (DEFAULT CURRENT_TIMESTAMP ON, UPDATE CURRENT_TIMESTAMP)

* practical: recommened DATETIME
- TIMESTAMP: support until 2038 year
- DATETIME support auto initialize/update
- 1bytes doesn't matter recently
- global service save time in UTC

* other type
- BLOB: binary large data. ex. audio, image, video
- ENUM: single selection type. select single value from list pre-defined. 
  - practical: commonly not used
- SET: multiple selection type. select multiple values from list pre-defined.
  - practical: commonly not used
