* summary
- DDL - table creation
  - pk: AUTO_INCREMENT
  - datetime: auto management - DEFAULT CURRENT_TIMESTAMP(+ ON UPDATE CURRENT_TIMESTAMP)
  - fk: child.parent_id -> parent.parent_id
  - ERD: table relationship overview
- DDL - table modification, deletion
  - structure change: ALTER TABLE ADD/MODIFY/DROP COLUMN
  - large amount modification: consider locking, speed
    - under 10,000 rows: fast enough
  - DROP TABLE: table deletion, TRUNCATE TABLE: data deletion. if FK exists, refuse command
  - ignore FK: SET FOREIGN_KEY_CHECKS=0; 
    - caution: immediately after, restore to 1
- DML
  - insert
    - 2types: all columns(1), specific columns(2 - recommended)
    - can insert multiple rows at once
    - can omit column with AUTO_INCREMENT, DEFAULT, NULLABLE
  - update
    - caution: with WHERE clause
    - MySQL Workbench: default @@SQL_SAFE_UPDATES=1
    - before update, check if data is correct using select
    - for bulk update, temporarily disable @@SQL_SAFE_UPDATES 
  - delete
    - caution: with WHERE clause
    - before delete, check if data is correct using select
    - delete slow, but conditional delete, can transaction rollback, not reset AUTO_INCREMENT
      - vs truncate: faster, but is not conditional, cannot transaction rollback, reset AUTO_INCREMENT
- constraint condition usage
  - NOT NULL: requirement
  - UNIQUE: prevent duplication
  - FK: value must exist in parent table
  - protect data integrity: validate constraint condition before insert, update, delete
