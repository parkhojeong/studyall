* RDBMS comparison
- Oracle
  - pros: very powerful, many function, best high stability. for finance domain
  - cons: expensive, require high expertise 
- MySQL
  - commonly used opensource RDBMS. big community
  - community edition, enterprise edition
  - easy to learn, use
  - cons: complex analyze query, large scale transaction performance
- PostgreSQL
  - open source RDBMS. standard SQL compliance, complex query process performance, extension function  
  - cons: small domestic user
- Microsoft SQL Server(MS SQL)
  - suitable for window server or .NET framework. high performance. provide variable tools
  - cons: proper to microsoft environment
- MariaDB
  - based on MySQL, compatible with MySQL protocol
- H2 Database(or SQLite)
  - embedded(in-memory) database, small footprint, easy to use
  - usage: development stage, mobile app internal store, small scale application

* MySQL

* practice - DB choice
- MySQL(MariaDB) or Oracle
1. candidate pool
2. cost, scale
   - large company/finance: Oracle. can handle cost, mission critical system for advanced function and technical support. 
   - small company/web service: MySQL. free. fast development and maintenance.
3. learn and development convenience
   - easy to learn.
   - many tools and library. high productivity
4. recent trend
   - save cost needs: free cost
   - open source performance and function improvement.
