* jpa execution flow
1. setup info read from META-INF/persistence.xml
2. EntityManagerFactory creation
3. EntityManager creation

* object and table mapping
- @Entity: object that jpa will manage
- @Id: pk mapping

* caution
- EntityManagerFactory is shared in application scope
- EntityManager must not be shared between threads
- jpa data change is executed in transaction

* jpql
- object oriented query language = sql to object
- query target is object
- sql abstraction is supported
- jpql does not depend on specific database
