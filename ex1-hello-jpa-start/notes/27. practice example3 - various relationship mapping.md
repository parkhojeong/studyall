* relationship practice 

* many-to-many must resolved 1:n + n:1  

* @JoinColumn property
- name: fk name
  - default: field name + "_" + pk column name
- referencedColumnName: target table pk column name
  - default: referenced table pk column name 
- foreignKey(only DDL): fk constraint
- same as @Column
  - unique
  - nullable
  - insertable, updatable
  - columnDefinition
  - table

* @ManyToOne main property
- optional: optionality of relationship
  - default: true
- fetch: global fetch strategy
  - default
    - @ManyToOne: EAGER
    - @OneToMany: LAZY
- cascade
- targetEntity
  - related entity type info setting
  - if a collection is used, type info can be known by generic type
  - commonly not used
