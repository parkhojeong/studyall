* pk mapping method
- manual assign: @Id
- auto generation: @GeneratedValue
  - IDENTITY: relies on db's auto-increment feature(supported by mysql, postgresql, sql server)
  - SEQUENCE: uses a db sequence(oracle, postgresql)
  - TABLE: key generator table
  - AUTO(default): choose the best strategy based on db 
  - UUID: uuid compliant with RFC 4122

* recommended pk strategy
- pk constraint: not null, unique, immutable
- use a surrogate key
- recommend: long type + surrogate key + identity/sequence/uuid strategy

* IDENTITY strategy 
- pk generation is delegated to db
- ex. mysql AUTO_INCREMENT
- id value is known after insert
- when em.persist(entity), immediately insert sql is executed and pk value is returned

* SEQUENCE strategy - @SequenceGenerator
- name(required): sequence generator name
- sequenceName: sequence name in db
  - default: hibernate_sequence
- initialValue(DDL only): 
  - default: 1
- allocationSize: increment size when the next pk value is requested
  - must same as db sequence increment size
  - default: 50(caution) 
- catalog, schema: db catalog and schema name

* @TableGenerator - TABLE generator strategy
